### SUGGESTIONS

**S1 — `toError()` not used in catch blocks (`src/observability/client.ts:33, 94`)**

Both catch blocks use `error instanceof Error ? error : new Error(String(error))` instead of `toError()` from `@/lib/errors`. CLAUDE.md is explicit about using `toError()`. The two are not equivalent: `toError()` extracts `.message` from non-Error objects and falls back to `JSON.stringify` before `String()`. Line 94 is inside the `if (rumEnabled)` block this PR significantly modified.

Fix:
```ts
import { toError } from '@/lib/errors';
.withError(toError(error)).error('Failed to initialize RUM');
```

**S2 — `match` function (lines 50–62) has zero test coverage**

The `match` function — which decides whether to inject trace headers into outgoing requests — has no tests. All 3 branches uncovered (CI confirmed: lines 50–62). The environment supports tests: file is already `@vitest-environment jsdom`, `window.location` is available. Extract `match` from `vi.mocked(datadogRum.init).mock.calls[0][0].allowedTracingUrls[0].match` and test: same-origin `/api/route`, cross-origin URL, `__nextjs` URL, malformed string.
