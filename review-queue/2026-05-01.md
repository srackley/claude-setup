### BLOCKING: 1

**B1 — First-wins guard broken when `experiment_key` is absent in API response**
- File: `packages/wanderu-app-tripsummary/src/components/TripSummary/TripSelectionSummary/hooks.ts`, lines 326–329
- The guard `feeExperimentKey === undefined` is intended to make the first trip's experiment data win for round-trip itineraries. However, if `feeMetadata.experiment` is truthy but `experiment_key` is absent from the API response, the assignment `feeExperimentKey = feeMetadata.experiment.experiment_key` sets `feeExperimentKey` to `undefined`. The guard then remains `true` on the next loop iteration, so the next leg's experiment data overwrites — the *last* leg wins rather than the first.
- Compare: the adjacent `fcVersion` guard (line 322) correctly checks `feeMetadata.version !== undefined` before assigning.
- **Fix:** Mirror the fcVersion pattern:
  ```ts
  if (feeMetadata.experiment?.experiment_key !== undefined && feeExperimentKey === undefined) {
      feeExperimentKey = feeMetadata.experiment.experiment_key;
      feeExperimentVariant = feeMetadata.experiment.variant_key;
  }
  ```
- **Challenger verdict:** Confirmed — no compensating guard exists downstream. For a round-trip where outbound has `experiment: { experiment_key: undefined }` and return has valid keys, the return trip's data fires the exposure event.

### SUGGESTIONS: 2

**S1 — No unit test for `useCanonicalTripsSelector` extraction logic**
- File: `packages/wanderu-app-tripsummary/src/components/TripSummary/TripSelectionSummary/hooks.ts`, lines 323–329
- Every downstream test that uses this hook mocks it entirely. The extraction code — reading `feeMetadata.experiment.experiment_key` and `variant_key` — is never executed by any test. If the API renames these fields, the exposure event silently stops firing with no test to catch it. CLAUDE.md: "TDD is non-negotiable."
- Missing test cases: (a) `fee_metadata.experiment` with valid keys → both fields extracted correctly; (b) experiment absent → both undefined; (c) first-trip-wins for round trips; (d) `feeMetadata` absent → both undefined.
- **Challenger verdict:** Confirmed — no such test exists anywhere in the codebase.

**S2 — `feeMetadata.experiment` access chain is fully untyped (`any`)**
- File: `packages/wanderu-app-tripsummary/src/components/TripSummary/TripSelectionSummary/hooks.ts`, lines 297–328
- `pricingData` originates from a `.js` selector with no type annotations. Every access — `feeMetadata.experiment.experiment_key` — is unchecked. A backend field rename silently produces `undefined` with no compiler error and no failing test.
- **Fix:** Add a local interface:
  ```ts
  interface FeeExperiment { experiment_key?: string; variant_key?: string; }
  const experiment = feeMetadata.experiment as FeeExperiment | undefined;
  ```
- **Challenger verdict:** Confirmed — `any` type confirmed via `selectLiveCheckPrice` in untyped `.js` file.
