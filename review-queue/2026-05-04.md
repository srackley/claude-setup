**Branch:** feat/ourbus-fee-names-932

### Summary

Splits OurBus fees (`usourBookingFee`, `usourFacilityFee`) into distinctly-named line items in the Trip Summary. Previously both displayed as "Carrier Fee". The fix adds `labelKeys: string[]` to fee objects and calls `t(fee.labelKeys, { defaultValue: t('carrier-fee') })` to resolve the first matching translation key or fall back to the generic label. Also renames PO translation keys from `ourbus-booking-fee` → `usourBookingFee` and `ourbus-facility-fee` → `usourFacilityFee` across 12 locale files to match the API fee IDs.

Closes #932. Replaces closed ui-react#931 and pservs#538/542.

Existing review: noahwelch's 4 suggestions (use `t()` for defaultValues) are all resolved and incorporated. One unresolved comment from eway-wanderu ("We really want to hard code these in the ui?") is outdated. fyeung86's unresolved comment about `carrier-fee` msgctxt answered below.

### Agent Findings

**code-reviewer:** (95%) hr-hr.po not migrated to new key names — all other 12 locales updated. (85%) `label: string` declared in both FeesProps interfaces but never read by either component. (82%) Component tests use `toHaveBeenCalledWith` (call signatures) rather than `screen.getByText` (rendered output); violates CLAUDE.md "test behavior not implementation".

**silent-failure-hunter:** Confirmed hr-hr.po issue — missingKeyHandler is a no-op so Croatian users silently see generic "Carrier Fee" with no log/error. `defaultValue: t('carrier-fee')` is unreachable dead code for carrier fees (labelKeys already ends in 'carrier-fee').

**type-design-analyzer:** `label` is redundant alongside `labelKeys` — both encode same binary classification; drift risk. `labelKeys: string[]` too loose — actual domain is `['platform-fee'] | [string, 'carrier-fee']`; tuple type would make i18next fallback intent explicit. `FeesProps` duplicated inline in both component files.

**pr-test-analyzer:** No `.skip`/`.only`/`.todo` markers — clean. Confirms component tests check call signatures not rendered behavior. Empty fees input / non-USD currency not tested in utils.test.ts.

**comment-analyzer:** fyeung86's question: `carrier-fee` does not need `msgctxt` — it is only a generic last-resort fallback; adding context would fragment the PO entry. Platform-fee `defaultValue: t('platform-fee')` also unreachable. PR's "Out of scope" claim for `TripLeg.tsx` in `wanderu-app-booking` is accurate.

### Challenger Verdicts

|---|---------|---------|
| 1 | hr-hr.po not migrated | **Confirmed** — `usourBookingFee`/`usourFacilityFee` absent from hr-hr.po; no i18next aliasing mechanism; Croatian users get generic Croatian "Carrier Fee" silently |

### Verdict

- `packages/wanderu-shellapp/src/translations/baseApp/hr-hr.po` — OurBus keys not migrated. Fix: rename `msgid "ourbus-booking-fee"` → `msgid "usourBookingFee"` and `msgid "ourbus-facility-fee"` → `msgid "usourFacilityFee"` (same strings, just the keys). One-line-per-key fix.

**Suggestions (1):**
- `labelKeys` should be typed as `['platform-fee'] | [string, 'carrier-fee']` instead of `string[]` — tuple makes i18next fallback intent explicit; no call-site breakage, no test fixture changes needed
