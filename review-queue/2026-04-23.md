`isLoading` is initialized with a lazy initializer: `useState(() => !hasFetched)`. If `hasFetched` is still `false` when the component mounts (loading = true) and then `hasFetched` flips to `true` before `experiment` is defined in Redux (the experiments slice populates asynchronously), `isLoading` is never set to `false` — the useEffect that drives `setIsLoading(false)` depends on `experiment`, which hasn't arrived yet. The hook permanently reports `isLoading: true` until the component unmounts and remounts. Race window: any component that renders during the experiment-fetch gap.  
*Evidence: Read hooks.ts; isLoading initialized with `() => !hasFetched`; useEffect deps array confirmed via file read.*
